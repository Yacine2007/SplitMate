L'invitation contient un code 2FA. La classe Portefeuille est liee 1:1 au Groupe. Paiement gere les transactions Stripe/CIB.

Diagramme de Sequence – Paiement partage

sequenceDiagram
    actor A as Ahmed
    actor B as Bilal
    actor C as Chaima
    participant App as SplitMate App
    participant Server as Serveur
    participant Pay as Passerelle de paiement (Stripe/CIB)

    Note over A,C: Membres du meme espace partage

    rect rgb(245, 243, 255)
        Note over A,Server: 1. Creation de la depense
        A->>App: Ajoute depense "Restaurant 9000 DA"
        App->>Server: POST /expenses {montant:9000, groupeId:"G1"}
        Server->>Server: Division 9000/3 = 3000 DA/pers.
        Server-->>App: Depense creee + parts
        App->>B: Notification : vous devez 3000 DA
        App->>C: Notification : vous devez 3000 DA
    end

    rect rgb(236, 252, 243)
        Note over B,Pay: 2. Paiement de Bilal
        B->>App: Consulte sa part (3000 DA)
        B->>App: Clique "Payer maintenant"
        App->>Pay: Requete paiement 3000 DA (Stripe/CIB)
        Pay-->>Server: Webhook confirmation paiement
        Server->>Server: Mise a jour statut
        Server-->>App: Paiement Bilal confirme
        App->>A: Notification : Bilal a paye
        App->>C: Notification : Bilal a paye
    end

    rect rgb(254, 243, 199)
        Note over C,Pay: 3. Paiement de Chaima
        C->>App: Consulte sa part (3000 DA)
        C->>App: Effectue le paiement
        App->>Pay: Requete paiement 3000 DA
        Pay-->>Server: Webhook confirmation
        Server-->>App: Paiement Chaima confirme
        App->>A: Notification : Chaima a paye
        App->>B: Notification : Chaima a paye
    end

    rect rgb(232, 240, 255)
        Note over Server: 4. Cloture
        Server->>Server: Toutes les parts sont payees
        Server->>Server: Mise a jour portefeuille commun
        App->>A: Depense reglee !
        App->>B: Depense reglee !
        App->>C: Depense reglee !
    end
Scenario complet : creation, division, notifications, paiements via Stripe/CIB et confirmation.

Diagramme d'Activite

flowchart TD
    Start([Debut]) --> Login[Connexion / Inscription]
    Login --> Has2FA{2FA activee ?}
    Has2FA -->|Oui| TwoFA[Saisir code 2FA]
    Has2FA -->|Non| Dashboard
    TwoFA --> CodeValid{Code valide ?}
    CodeValid -->|Oui| Dashboard[Tableau de bord]
    CodeValid -->|Non| TwoFA

    Dashboard --> Choice{Action ?}

    Choice -->|Ajouter amis| Invite[Envoyer invitation avec code 2FA]
    Invite --> Friend2FA{Verification du code par l'ami}
    Friend2FA -->|Accepte| SharedSpace[Espace partage cree]
    Friend2FA -->|Refuse| Invite
    SharedSpace --> Dashboard

    Choice -->|Nouvelle depense| SelectGroup[Selectionner groupe]
    SelectGroup --> EnterAmount[Saisir montant + description]
    EnterAmount --> SelectCat[Choisir categorie]
    SelectCat --> AutoSplit[Division automatique]
    AutoSplit --> NotifyMembers[Notifier tous les membres]
    NotifyMembers --> ViewShares[Chaque membre voit sa part]

    ViewShares --> PayNow{Payer maintenant ?}
    PayNow -->|Oui| OnlinePay[Paiement en ligne via Stripe/CIB]
    OnlinePay --> ConfirmPay{Confirme ?}
    ConfirmPay -->|Oui| UpdateWallet[Mise a jour portefeuille]
    ConfirmPay -->|Non| Retry[Reessayer]
    Retry --> OnlinePay
    PayNow -->|Plus tard| Pending[En attente]
    UpdateWallet --> AllPaid{Tous ont paye ?}
    AllPaid -->|Oui| Close[Depense cloturee]
    AllPaid -->|Non| ViewShares
    Pending --> Close
    Close --> End([Termine])

    Choice -->|Explorer| Browse[Parcourir activites]
    Browse --> Filter[Filtrer (restaurant, cinema...)]
    Filter --> SelectPlace[Choisir un lieu]
    SelectPlace --> SelectGroup
